# test1.py 程式說明

## 概述

這是一個 **Open WebUI 的 Filter 插件**範例。Filter 插件讓你可以在對話流程的兩個關鍵時間點介入：

- **inlet**：使用者訊息送出前
- **outlet**：AI 回應回傳前

---

## 架構說明

```
Filter
├── Valves        ← 管理員設定（全域）
├── UserValves    ← 使用者設定（個人）
├── __init__()    ← 初始化
├── inlet()       ← 入口處理器（請求前）
└── outlet()      ← 出口處理器（回應後）
```

---

## 各元件說明

### Valves（管理員設定）

由系統管理員配置，套用到所有使用者。

| 欄位 | 預設值 | 說明 |
|------|--------|------|
| `priority` | `0` | 過濾器執行優先順序，數字越小越先執行 |
| `max_turns` | `8` | 系統允許的最大對話輪數上限 |

### UserValves（使用者設定）

每個使用者可以自行調整的設定。

| 欄位 | 預設值 | 說明 |
|------|--------|------|
| `max_turns` | `4` | 該使用者允許的最大對話輪數 |

---

## 核心邏輯：對話輪數限制

這個 Filter 的主要功能是**限制對話輪數**，防止對話過長。

```python
max_turns = min(__user__["valves"].max_turns, self.valves.max_turns)
```

實際上限 = 使用者設定值 與 系統設定值 兩者取較小的，確保使用者無法繞過系統限制。

**範例：**
- 系統上限（Valves.max_turns）= 8
- 使用者上限（UserValves.max_turns）= 4
- 實際上限 = min(4, 8) = **4 輪**

若訊息數量超過上限，會拋出例外並阻止請求：
```
Conversation turn limit exceeded. Max turns: 4
```

---

## 流程圖

```
使用者送出訊息
      ↓
  [ inlet() ]  ← 檢查對話輪數，超過則拋出例外
      ↓
  AI 處理請求
      ↓
  [ outlet() ] ← 目前直接回傳，可在此加入後處理邏輯
      ↓
回應顯示給使用者
```

---

## 擴充建議

- 在 `inlet()` 加入關鍵字過濾或敏感詞檢查
- 在 `outlet()` 加入回應內容的日誌記錄或分析
- 啟用 `self.file_handler = True` 來自訂檔案上傳的處理邏輯
